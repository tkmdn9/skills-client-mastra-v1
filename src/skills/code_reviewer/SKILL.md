# コードレビュースキル

以下の観点でコードをレビューしてください：

## レビュー観点

1. **正確性** — ロジックのバグ、エッジケースの漏れ
2. **セキュリティ** — インジェクション、認証・認可の問題
3. **パフォーマンス** — 不必要なループ、N+1問題
4. **可読性** — 命名、関数の責務分離
5. **テスト** — テストカバレッジの不足

## 出力形式

```
### 重大 🔴
- {問題の説明} (行番号)

### 警告 🟡
- {問題の説明} (行番号)

### 提案 🟢
- {改善提案} (行番号)
```

重大な問題がない場合は「問題なし」と明記する。
